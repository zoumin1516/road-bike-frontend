# 公路车资料站：当前已完成功能 + 下一阶段开发计划

## 一、项目当前定位

当前项目已经从早期的“公路车天梯 / 评分系统”方向，收敛为一个更务实、更容易持续建设的产品：

**公路车品牌、车型、配置、零部件的结构化查询展示网站。**

这意味着当前版本的核心目标已经明确为：

- 让用户可以按品牌、车型、配置、零部件四个维度浏览数据
- 让用户可以通过筛选快速缩小范围
- 让用户可以进入详情页查看结构化资料
- 让网站整体观感更接近正式上线的产品资料站，而不是 demo

## 四、下一阶段开发计划

下面这部分是建议的下一阶段推进路线，我后续可以直接按这个顺序继续做。

## Phase 2：产品能力增强

### 1. 搜索页升级为真正的全站检索入口

当前搜索页存在，但还可以明显增强。

下一步建议：

- 支持跨品牌 / 车型 / 配置 / 零部件联合搜索
- 支持搜索结果分类展示
- 支持搜索词高亮
- 支持附带筛选项
- 支持空结果提示与纠错建议

目标：

把搜索页从“搜索功能页”升级成真正的全站检索中心。

---

### 2. 详情页参数区升级为更强的 specs 结构

当前详情页已经具备统一布局，但参数块仍有进一步提升空间。

下一步建议：

- 将参数内容改成更明确的 key-value grid
- 对配置页、零部件页引入更结构化的 specs table
- 对可量化字段做更醒目的数字卡片
- 对链接、ID、状态等字段做更明确的区分显示

目标：

让详情页更像专业器材数据库，而不是视觉升级后的普通详情页。

---

### 3. 已生效筛选项展示条

当前筛选已经可用，但用户对“自己现在筛了什么”感知还可以更强。

下一步建议：

- 在列表页顶部显示 active filters 条
- 每个筛选项支持单独取消
- 支持“一键清空全部筛选”
- 支持结果数量和筛选状态联动显示

目标：

降低复杂筛选场景下的认知负担。

---

### 4. 排序能力

当前列表页已有分页和筛选，但缺少排序能力。

下一步建议：

- 品牌页：按名称排序
- 车型页：按名称 / 发布年份 / 在售状态排序
- 配置页：按价格 / 年份排序
- 零部件页：按价格 / 品牌 / 类别排序

目标：

让列表页更像真正的数据检索页面。

---

### 5. 空结果态与异常态优化

当前功能主路径基本可用，但边界态还可继续产品化。

下一步建议：

- 无结果时显示清晰提示
- 提供返回全部 / 清空筛选按钮
- 接口异常时给出更友好的展示
- 对详情页缺失字段做更统一的空值展示规则

目标：

提升产品完整度与稳定感。

---

## Phase 3：数据库产品化增强

### 6. 卡片视图 / 表格视图切换

下一步建议：

- 列表页支持 card / table 切换
- 表格模式下显示更多字段
- 支持更高密度浏览
- 为未来批量比对能力做准备

目标：

让网站更接近长期使用的数据库工具。

---

### 7. 对比能力

这是后续非常自然的一步。

下一步建议：

- 车型对比
- 配置对比
- 零部件对比
- 基于关键参数生成对比表

目标：

把资料站进一步升级为决策辅助工具。

---

### 8. 收藏 / 最近浏览 / 历史记录

如果后续希望提升长期使用价值，可以引入轻量状态能力。

下一步建议：

- 收藏车型 / 配置 / 零部件
- 最近浏览记录
- 本地浏览历史
- 快速回看机制

目标：

增强复访与长期使用场景。

---

### 9. 数据完善与字段扩容

随着网站基础结构稳定，后续最重要的增长点将是数据质量本身。

下一步建议：

- 扩大品牌覆盖
- 扩大车型覆盖
- 增加 build 细节字段
- 增加 component 细节字段
- 明确各字段来源和可信度

目标：

让网站从“结构好看”真正走向“数据足够有价值”。
